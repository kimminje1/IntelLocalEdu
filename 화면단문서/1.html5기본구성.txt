*DTD(Document Type Definition)

이 문서는 정해놓은 규칙에 따라 마크업한다 라는 의미로
다양한 브라우저들에게 마크업에
대한 일관된 화면표시가 될 수 있도록 해준다
 
*태그(Tag) 
<p>,<a>등 과 같이 명령어의 형태가 <>(각 괄호)로 되어 있는 것을
태그라고 한다

<body></body>와 같이 태그의 시작과 종료를 하나의 요소(Element)라고 한다

이러한 요소들을 이용하여 웹 문서를 작성하는 것을 마크업(markup)이라고 한다

*head태그에 사용할 수 있는 태그들
요소명			설명
meta			html 문서의 설명, 키워드, 문서의 작성자 등을 지정한다	
title			html 문서의 제목을 지정한다
link			외부 CSS파일을 HTML 문서에 연결할때 사용한다
style			html 문서내에서 스타일을 정의한다
script			자바스크립트르를 html 문서내부에서 정의하거나
				외부에서 자바스크립트 파일을 불러올 때 사용한다
				
<meta charset="UTF-8">처럼 태그 내부에 부가적인 설정값을 선언하는 것을
속성(Attribute)라고 한다	

*마크업 기본 문법 규칙
1.태그는 시작태그와 종료 태그가 있어야 한다
<p>ㅁㅁㅁ</p>	  o
<p>ㅁㅁㅁ       x
<br>		  x	
<br/>		 o

2.태그는 제대로 중첩되어야 한다
<p><strong>ㅁㅁㅁ</strong>태그는 제대로 중첩되어야 한다</p>	  o
<p><strong>ㅁㅁㅁ</p>	태그는 제대로 중첩되어야 한다</strong>    x

3.요소 및 속성 이름은 소문자여야 한다
<p><a href="local">ㅁㅁㅁ</a>요소 및 속성 이름은 소문자 여야 한다</p>	  o
<p><a HREF="local">ㅁㅁㅁ</a>요소 및 속성 이름은 소문자 여야 한다</p>	  x

4.img 태그에는 alt속성이 있어야 한다
<img src = "images/today.gif" alt="오늘">  o
<img src = "images/today.gif">  x

 *블록 요소(Element) 특징
 1.블록 태크는 줄 바꿈이 일어난다
 2.텍스트와 인라인 요소를 자식 요소로 포함할 수 있다
 3.블록 태그 중에는 블록 태크를 자식 태그로 포함할 수 있는 태그와
 포함할 수 없는 태그가 잇다
 
 *인라인 태그 특징.
 1.줄 바꿈이 없다
 2.텍스트와 인라인 태그를 자식 태그로 포함할 수 있다 
 3.인라인 태그는 블록 태그로 자식 태그로 포함할 수 있다
