==클래스 간의 관계(relation)====

객체지향의 3요소:상속, 다형성, 캡슐화

상속(Inheritance)-구현

기존의 클래스를 재사용하여 새로운 클래스를 작성하는 것

상속을 통해서 클래스를 작성하면 보다 적은 양의 코드로 기존의 클래스를 쉽게
보다 나은 클래스로 작성할 수 있고 코드를 공통적으로 관리할 수 있기 때문에
코드의 추가 및 변경이 용이하다

이러한 특징은 코드의 재사용성을 높이고 코드의 중복을 제거한다

조상 클래스 - 부모(parent) 클래스, 상위(super)클래스, 기반(base) 클래스
자손 클래스 - 자식(child) 클래스, 하위클래스(sub), 파생된(derived) 클래스

상속은 단일 상속만 가능하다 ->여러부모를 가질수 없다

모든 클래스의 조상 Object 클래스
최상위 클래스라고 부른다. 혹은 root

상속 키워드 extends
-생성자와 초기화 블럭은 상속되지 않는다. 멤버(member)만 상속된다.
-자식 클래스의 맴버 개수는 부모 클래스보다 항상 같거나 많다

==포함관계(composite)==
클래스의 멤버변수로 다른 클래스 타입의 참조변수가 선언되어 있는 것
